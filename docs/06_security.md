# Pixel-Link 보안 및 링크 악용 방지 대책서

단축 URL 서비스는 피싱, 스팸, 불법 도박 사이트 등의 통로로 악용되기 매우 쉽습니다. 서비스 제공자의 도메인이 유해 사이트로 분류되어 브라우저 경고 창이 뜨게 되면 비즈니스 전체가 마비됩니다. 
따라서 본 문서에서는 Pixel-Link 상용 빌드 단계에서 반드시 반영해야 할 보안 및 개인정보 보호 방안을 기술합니다.

---

## 1. 유해 링크 사전 차단 정책 (Anti-Spam & Phishing)

사용자가 새로운 링크를 생성할 때 (`POST /api/links`), 백엔드는 목적지 URL을 즉시 검사하여 안전성을 보장해야 합니다.

### 1.1. Google Safe Browsing API 연동
* **기능**: 링크 등록 요청 시 목적지 URL을 Google Safe Browsing API에 전송하여 악성코드, 피싱, 유해 도메인 여부를 판단합니다.
* **대응**: 위험 도메인으로 확인되는 즉시 등록을 거부하고 에러 메시지를 반환합니다. (`code: MALICIOUS_URL`, 상태 코드 `400`)

### 1.2. 도메인 블랙리스트 및 키워드 필터링
* 내부 데이터베이스나 캐시(Redis 등)에 차단할 악성 키워드와 블랙리스트 도메인을 관리합니다.
* 예: `*.top`, `*.xyz`, `bit.ly` 우회 링킹, 성인/도박 키워드가 포함된 주소는 원천적으로 차단합니다.

---

## 2. API 및 리다이렉션 보호 (Rate Limiting & DoS Protection)

악의적인 사용자가 봇(Bot)을 이용해 API를 무차별 호출하거나, 특정 리다이렉트 주소에 디도스(DDoS) 형태의 부하를 주어 서버를 다운시키는 것을 방지합니다.

### 2.1. API 요청 제한 (API Rate Limiting)
* Spring Boot 내에서 **Bucket4j** 라이브러리 또는 Spring Security Filter 단에서 IP별 처리 한계를 지정합니다.
* **제한 스펙**:
  * 단축 링크 생성 API: 동일 IP 기준 **분당 최대 30회** 제한.
  * 대시보드 로그인/조회 API: 동일 IP 기준 **분당 최대 100회** 제한.

### 2.2. 리다이렉트 폭주 제어
* 특정 단축 링크 슬러그로 무차별 접속(매크로 봇)이 감지되면 서버 자원 낭비를 줄이기 위해 해당 IP에 대해 일시적으로 429 Too Many Requests 응답을 전달합니다.

## 3. 개인정보 보호 및 법적 준수 (Privacy Protection)

마케팅 픽셀 트래킹은 유저의 유입 경로 정보를 식별해야 하지만, 동시에 개인정보보호법(GDPR, 국내 개인정보보호법)을 명확히 준수해야 합니다.

### 3.1. IP 주소 단방향 암호화 (IP Hash)
* 중복 클릭 방지(Click Fraud) 분석을 위해 기기 판별용 IP 주소가 필요하지만, 원본 IP 주소를 그대로 DB에 적재하면 안 됩니다.
* **해결책**: 백엔드 수집 시점에 **`IP 주소 + 당일 날짜 솔트(Salt)`** 값을 **SHA-256**으로 단방향 해싱하여 `ip_hash` 컬럼에 저장합니다.
* **효과**: 원본 IP 복구가 불가능하여 개인정보 침해 소지가 없고, 당일 동일 인물의 중복 어뷰징 클릭은 해시 일치 여부로 식별이 가능합니다.

## 4. XSS (Cross-Site Scripting) 방지

1단계에서 제공하는 마케터 전용 `customScript` 기능은 잘못 구현할 경우 타 사용자의 대시보드를 변조하거나 악성 스크립트를 삽입(XSS)하는 용도로 활용될 수 있습니다.

### 4.1. 권한 세분화
* `customScript` 기능은 무료 사용자는 절대 쓸 수 없도록 차단하며, 본인 인증이 완료된 **Premium(유료) 요금제 가입자에게만 활성화**합니다.

### 4.2. 실행 격리 및 입력 검증
* 대시보드 화면(`dashboard.html`) 내부에서는 어떠한 경우에도 사용자가 입력한 커스텀 스크립트가 그대로 실행되지 않도록 Thymeleaf에서 텍스트 탈출(Escape, `th:text`) 처리를 합니다.
* 유저가 접속하는 리다이렉션 페이지(`redirect.html`)에서만 안전하게 제한된 샌드박스 스크립트 형태로 렌더링(`th:utext`)되도록 관리자 모니터링 시스템을 구축합니다.
