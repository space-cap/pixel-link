# Pixel-Link 설계 문서 패키지 Index

본 디렉토리는 Pixel-Link 상용 플랫폼의 비즈니스 설계, 기능 구현 계획, 데이터 구조, API 명세, UI/UX 규격, 보안 대책 및 셋업 가이드를 모아둔 설계 문서 보관소입니다.

프로젝트의 전체 맥락과 아키텍처를 원활하게 파악할 수 있도록 아래 **논리적 순서**에 따라 문서를 읽어보시는 것을 강력히 권장합니다.

---

## 📚 문서 읽기 권장 순서 (Step-by-Step)

### [01. 수익화 및 구독 요금제 명세서 (01_monetization.md)](file:///h:/lee/pixel-link/docs/01_monetization.md)
* **내용**: 돈을 벌기 위한 비즈니스 모델 설계도입니다. Free/Starter/Premium 구독 등급별 권한 제한 규칙, 3단계 광고 수익 배분율, 4단계 페이월 수수료 구조를 규정합니다.
* **왜 먼저 보나요?**: 서비스의 비즈니스 규칙이 시스템 권한 제어와 결제 아키텍처의 근간이 되기 때문입니다.

### [02. 전체 로드맵 및 1단계 구현 계획서 (02_implementation_plan.md)](file:///h:/lee/pixel-link/docs/02_implementation_plan.md)
* **내용**: 1단계(마케팅 픽셀)부터 4단계(페이월)까지의 기술 성장 전략과 1단계 상용화 개발을 위한 Spring Boot 구조적 설계 로드맵을 담고 있습니다.
* **왜 보나요?**: 비즈니스를 어떻게 점진적인 단계를 밟아 릴리즈해 나갈지 로드맵을 명확히 설명해 줍니다.

### [03. 데이터베이스 상세 명세서 (03_db_schema.md)](file:///h:/lee/pixel-link/docs/03_db_schema.md)
* **내용**: 회원, 단축 링크, 스마트 라우팅 규칙, 클릭 로그, 정산 테이블의 세부 칼럼 정보와 SQLite(로컬) / PostgreSQL(상용) 이중화 전략을 다룹니다.
* **왜 보나요?**: 데이터가 시스템 안에서 어떻게 흘러가고 저장되는지 백엔드 뼈대를 이해할 수 있습니다.

### [04. REST API 규격서 (04_api_spec.md)](file:///h:/lee/pixel-link/docs/04_api_spec.md)
* **내용**: 링크 생성/수정/삭제/조회 및 통계 데이터를 다루는 HTTP 요청/응답 JSON 포맷과 예외 처리 에러 코드 체계입니다.
* **왜 보나요?**: 대시보드 화면(프론트엔드)과 스프링 부트(백엔드) 간의 통신 약속을 규정합니다.

### [05. UI/UX 디자인 가이드라인 (05_ui_design.md)](file:///h:/lee/pixel-link/docs/05_ui_design.md)
* **내용**: 프리미엄 다크 테마(HSL), 글래스모피즘 스펙, 폰트(Inter/Outfit), 대시보드 및 리다이렉트 대기 화면 레이아웃과 호버/복사 인터랙션 가이드입니다.
* **왜 보나요?**: 사용자에게 뛰어난 사용성과 프리미엄한 첫인상을 주기 위한 화면 규격입니다.

### [06. 보안 및 링크 악용 방지 대책서 (06_security.md)](file:///h:/lee/pixel-link/docs/06_security.md)
* **내용**: 도메인 악성 차단(Google Safe Browsing 연동), API 요청 제한(Rate Limiting), IP 해싱 개인정보 준수, 스크립트 보안 지침을 정리합니다.
* **왜 보나요?**: 서비스 도메인이 블랙리스트에 등록되어 사이트가 차단되는 리스크를 원천 예방합니다.

### [07. 환경 구성 및 실행 가이드 (07_setup_guide.md)](file:///h:/lee/pixel-link/docs/07_setup_guide.md)
* **내용**: Java/Gradle 빌드 명령, 로컬 환경에서 SQLite 자동 구성 실행법, 상용 환경 배포 및 PostgreSQL 환경 변수 주입 방법 등을 다룹니다.
* **왜 보나요?**: 실제 개발 프로젝트를 가동하고 서버에 배포하여 구동하기 위한 기술적 지침서입니다.

### [08. 다국어(i18n) 지원 및 확장 가이드 (08_i18n_guide.md)](file:///h:/lee/pixel-link/docs/08_i18n_guide.md)
* **내용**: 한국어, 영어, 일본어 3개 국어 번들 로드 방식과 향후 추가적인 글로벌 언어 팩 확장(properties 추가) 가이드를 제공합니다.
* **왜 보나요?**: 글로벌 상용 서비스로서 다국어 지원을 어떻게 처리했고 향후 새로운 언어를 어떻게 추가하는지 이해하는 지침서입니다.

### [09. 비회원 체험 기능 및 마케팅 랜딩 페이지 구현 계획 (09_anonymous_landing_plan.md)](file:///h:/lee/pixel-link/docs/09_anonymous_landing_plan.md)
* **내용**: 로그인 없는 1초 단축 기능, 마케팅 유도 랜딩 페이지, 설정값 실시간 조정 기능 및 스케줄러 청소 메커니즘을 상세히 다룹니다.
* **왜 보나요?**: 체험 유입 강화와 리소스 제어(만료일)라는 두 가지 마케팅적/기술적 목표를 어떻게 달성하는지 아키텍처를 보여줍니다.
