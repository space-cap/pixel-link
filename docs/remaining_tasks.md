# Pixel-Link 프로덕션 출시를 위한 잔여 및 고도화 작업 로드맵

본 문서는 1단계부터 4단계까지의 핵심 기능(비회원 단축, 스마트 라우팅, 광고 수익 정산, 페이월 콘텐츠 잠금) MVP 개발 완료 이후, **실제 상용 서비스(Production)로 배포하고 마케터 대상 비즈니스를 개시하기 위해 필요한 잔여 개발 및 고도화 항목**을 정리한 로드맵입니다.

## 🔍 상세 작업 명세서

### 1. 보안 및 실제 로그인 인증 체계 도입 (Security & OAuth2)
- **현 상태**: 대시보드 및 API 접근 시 `userId=test-user`와 같은 쿼리 파라미터 기반의 모의 계정으로 동작하고 있습니다.
- **필수 개발**:
  - **Spring Security** 프레임워크를 연동하여 인증되지 않은 유저의 대시보드(/dashboard) 접근을 차단합니다.
  - 마케터들이 번거로운 가입 절차 없이 쉽게 가입하도록 **소셜 로그인 (Google, Kakao, Naver OAuth2)**을 연동합니다.
  - 가입 즉시 데이터베이스에 사용자 테이블 레코드를 자동으로 영적(Persistent) 생성하고 세션 혹은 JWT 토큰을 발급하여 로그인 상태를 유지시킵니다.

### 2. 모의 PG 결제를 실제 PG 연동으로 고도화 (Payment Integration)
- **현 상태**: 페이월 화면(`paywall.html`)에서 [결제 완료] 모의 버튼 클릭 시 AJAX를 통해 강제 승인 처리하고 있습니다.
- **필수 개발**:
  - **토스페이먼츠(Toss Payments)** 또는 **포트원(PortOne - 옛 아임포트)** SDK를 웹뷰에 이식합니다.
  - 백엔드에 결제 승인 API 엔드포인트(`/api/payments/confirm`)를 연동하여 카드사 인증 및 최종 결제를 동기 승인 처리합니다.
  - 네트워크 단절 및 비정상 이탈 시 거래 정합성을 유지하기 위해 PG사로부터 결제 결과를 전달받는 **Webhook 수신용 API**를 개발하여 이중 잠금 처리합니다.

### 3. 단축 전용 숏 도메인(Short Domain) 및 HTTPS 보안 연결 설정
- **현 상태**: 단축 URL이 `http://localhost:8090/{code}`로 생성되어 로컬 서버에서만 작동합니다.
- **필수 개발**:
  - 마케팅 링크의 스팸 차단을 막기 위해 전용 숏 도메인(예: `pxl.li`, `pxlk.to` 등)을 구입하여 도메인을 바인딩합니다.
  - 단축 주소로 유입되는 모든 트래픽을 처리하는 별도의 리다이렉트 전용 프록시(Nginx 등) 및 **SSL 인증서(HTTPS)** 설정을 진행합니다.
  - 백엔드 `getBaseUrl` 로직을 설정 환경 변수 파일(`application.properties`)에서 주입받아 동적으로 실제 숏 도메인 주소로 발급하게 조율합니다.

### 4. 출금 및 정산 관리자 백오피스 개발 (Admin Settlement Approval)
- **현 상태**: 사용자가 출금 신청 시, DB 정산 기록 잔액을 마이너스 처리해 차감하고 PENDING 상태로 보관하는 백엔드 로직까지만 구현되어 있습니다.
- **필수 개발**:
  - 관리자(`userId=admin` 혹은 admin 역할을 가진 계정)가 로그인했을 때 진입하는 **정산 승인 관리 대시보드**를 개발합니다.
  - 대시보드에서 신청 유저의 은행 정보, 계좌번호, 예금주명, 신청 금액을 한눈에 조회합니다.
  - 수동으로 계좌 이체를 처리한 후, 관리자가 `[출금 완료]` 버튼을 누르면 해당 정산 신청 상태를 `COMPLETED`로 수정하고 거래를 확정하는 어드민 전용 API를 구현합니다.

### 5. 데이터베이스 이식 및 형상 관리 (PostgreSQL & Flyway)
- **현 상태**: 개발의 편의성을 위해 로컬 단일 파일 DB인 SQLite (`pixel-link-local.db`)를 주 데이터베이스로 사용 중입니다.
- **필수 개발**:
  - 트래픽 밀림 현상 방지 및 대규모 데이터 분산 처리를 위해 상용 DB인 **PostgreSQL (혹은 MySQL)** 데이터베이스로 완전히 전환합니다.
  - 테이블 스키마 DDL 변경 관리를 위해 스프링 부트에 **Flyway** 마이그레이션 라이브러리를 연동하여, 로컬 환경에서 적용된 모든 마이그레이션 히스토리(`V1__init.sql`, `V2__add_routing_rules.sql` 등)를 운영 DB 구축 시 자동 순차 배포되도록 구성합니다.

### 6. FAQ 및 공지사항 게시판 개발
- **현 상태**: 대시보드 메인 외에 FAQ나 공지사항을 확인할 수 있는 별도의 고객 센터 게시판이 부재합니다.
- **필수 개발**:
  - 마케터들이 자주 묻는 질문(FAQ)을 쉽고 빠르게 해결할 수 있는 아코디언식 자주 묻는 질문 보드를 구성합니다.
  - 서비스 점검 사항 및 정책 변경 내용을 알릴 수 있는 조회 전용 공지사항(Notice) 게시판 화면을 개발합니다.
  - 관리자가 어드민 센터에서 공지사항을 직접 등록/수정/삭제할 수 있는 간단한 어드민 기능과 권한 제어 장치를 추가합니다.

### 7. 1:1 문의 및 제휴 문의 기능 구현 (Support & Partnership Requests)
- **현 상태**: 사용자가 고객 센터를 통해 다이렉트로 문의하거나 비즈니스 제휴 제안서를 송신할 접수 채널이 없습니다.
- **필수 개발**:
  - 기술적 결함이나 요금 결제 실패 건에 대해 사용자가 문의글을 업로드할 수 있는 **1:1 문의 채널**을 개발합니다.
  - 외부 광고주 또는 제휴를 희망하는 마케터가 정식 비즈니스 제안을 송신할 수 있는 **제휴 문의 폼**을 구현합니다.
  - 문의글 접수 시 데이터베이스에 정렬 저장하고, 담당자 관리용 메일 주소로 즉각 알림을 전송하는 SMTP 메일 발송 장치를 구축합니다.

### 8. 구글 애드센스 등 실제 광고 네트워크 탑재
- **현 상태**: 광고가 활성화된 단축 링크 방문 시, 임의의 모의 마케팅 배너 이미지와 5초 대기 화면만 로드됩니다.
- **필수 개발**:
  - **Google AdSense** 또는 기타 서드파티 광고 네트워크(Ad Network)의 스크립트 연동을 완료합니다.
  - 광고 배너 영역에 광고 스크립트 코드(`<script async src=...></script>`)를 탑재하여 실제 마케팅 수익이 링크 소유자에게 정상 귀속될 수 있는 구도를 마련합니다.

### 9. 대시보드 통계 시각화 고도화 (Analytics UI Visual)
- **현 상태**: 접속 로그 수치가 텍스트 리스트 형태로만 모달에 표시되어 직관적인 데이터 분석이 어렵습니다.
- **필수 개발**:
  - 대시보드 웹 뷰에 **Chart.js** 또는 **Recharts** 같은 자바스크립트 차트 라이브러리를 임포트합니다.
  - 날짜별 클릭 수 추이(라인 차트), 기기별/OS별 점유율(도넛 차트), 유입 국가 분포(지도 또는 바 차트)를 그라데이션이 적용된 세련된 디자인으로 시각화 연출합니다.
