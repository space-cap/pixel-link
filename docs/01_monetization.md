# Pixel-Link 수익화 및 구독 요금제 명세서

본 문서는 Pixel-Link 서비스의 수익 모델과 비즈니스 로직을 규정합니다. 설계된 모델은 1단계(SaaS 구독)부터 3단계(광고 수익 분배), 4단계(페이월 수수료)까지 유기적으로 확장되며, 백엔드 코드의 회원 등급별 권한 검증 로직의 기준이 됩니다.

---

## 1. 구독 요금제 (SaaS Subscription) 구조 (1~2단계 중심)

사용자는 매달 일정한 구독료를 지불하고 등급에 맞추어 단축 URL 및 픽셀 기능을 제공받습니다.

| 요금제 명칭 | 월 구독료 (VAT 포함) | 생성 가능 링크 수 | 제공 기능 및 한계 | 타겟 고객 |
| :--- | :--- | :---: | :--- | :--- |
| **FREE (무료)** | 0원 | 최대 3개 | - 기본 단축 리다이렉션만 제공<br>- 마케팅 픽셀(FB/GA) 삽입 **불가**<br>- 맞춤형 주소(Custom Slug) **불가** | 일반 개인 유저 |
| **STARTER (스타터)** | 19,000원 | 최대 100개 | - **[1단계]** 마케팅 픽셀 삽입 기능 활성화<br>- 링크당 최대 2개 픽셀 등록 가능<br>- 맞춤형 주소(Custom Slug) 허용 | SNS 개인 셀러, 마케터 |
| **PREMIUM (프리미엄)** | 49,000원 | 무제한 | - **[2단계]** 스마트 라우팅 및 딥링크 무제한 제공<br>- 링크당 무제한 픽셀/커스텀 스크립트 허용 | 전문 커머스 셀러, 대행사 |

### 1.1. 백엔드 등급 검증 로직 (1단계 적용)
* 링크 생성 API (`POST /api/links`) 호출 시, 요청 회원의 `subscription_tier`를 조회합니다.
* 무료 회원이 4번째 링크를 만들려고 하거나 픽셀 ID를 포함하여 요청하는 경우 `400 Bad Request` 에러를 반환합니다. (`code: LIMIT_EXCEEDED` 또는 `code: FEATURE_LOCKED`)

## 2. 광고 수익 쉐어 모델 (3단계: Ad Short Links)

대량의 트래픽을 유발하는 유저(커뮤니티 업로더, 인플루언서 등)가 광고가 노출되는 링크를 생성하고, 발생한 수익을 플랫폼과 나눠 갖는 구조입니다.

### 2.1. 작동 프로세스
1. 유저가 링크 생성 시 `isAdEnabled = true` 옵션을 선택합니다.
2. 링크 접속 시 5초 대기 화면(`adTimerSeconds = 5`)에 구글 애드센스 등 플랫폼이 지정한 광고 영역이 노출됩니다.
3. 광고 노출 및 클릭 정보가 `click_logs` 테이블에 적재됩니다.
4. 백엔드 배치(Batch) 엔진이 스팸성 어뷰징 클릭을 필터링한 후 정산 수익을 집계합니다.

### 2.2. 정산 및 분배 비율
* **수익 배분 비율**: **플랫폼 30% : 링크 생성자 70%**
* **정산 프로세스**:
  * 유저의 누적 정산금액이 **10,000원 이상**일 때 출금 신청이 가능합니다.
  * 정산 신청 시 `settlements` 테이블에 `PENDING` 상태로 등록되며, 매월 10일 관리자 승인 후 유저 계좌로 일괄 지급(`COMPLETED`) 처리됩니다.

## 3. 페이월 콘텐츠 잠금 수수료 모델 (4단계: Paywall & Locks)

디지털 파일(PDF, 쿠폰 번호, 비공개 주소 등)을 링크 뒤에 잠금 처리해두고 결제 완료자에게만 리다이렉트해 주는 기능입니다.

### 3.1. 요금 구조 및 결제 흐름
1. 링크 소유자가 결제 금액(`price`)을 지정하여 페이월 링크를 생성합니다. (`isPaywalled = true`)
2. 다른 접속자가 해당 단축 링크 접속 시, 중간 페이지에 토스페이먼츠 등의 PG 결제 위젯이 뜹니다.
3. 결제가 성공하면 결제 확인 토큰이 발급되고, 세션이 인증된 접속자에 한해서만 백엔드가 목적지 URL로 최종 리다이렉트합니다.

### 3.2. 플랫폼 중개 수수료
* **수수료율**: **총 결제 금액의 5%** (결제 대행사 PG 수수료 약 3% 내외는 링크 소유자 부담)
* **정산 흐름**:
  * 결제 건당 5%의 수수료를 즉시 공제한 후, 나머지 금액이 링크 소유자의 플랫폼 예치금으로 쌓입니다.
  * 예치금은 주간 단위 혹은 월간 단위로 소유자가 등록한 정산용 계좌로 지급됩니다.
