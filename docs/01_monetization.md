# Pixel-Link 수익화 및 구독 요금제 명세서

본 문서는 Pixel-Link 서비스의 수익 모델과 비즈니스 로직을 규정합니다. 설계된 모델은 1단계(SaaS 구독)부터 3단계(광고 수익 분배), 4단계(페이월 수수료)까지 유기적으로 확장되며, 백엔드 코드의 회원 등급별 권한 검증 로직의 기준이 됩니다.

---

## 1. 구독 요금제 (SaaS Subscription) 구조 (1~2단계 중심)

사용자는 매달 일정한 구독료를 지불하고 등급에 맞추어 단축 URL 및 픽셀 기능을 제공받습니다.

### 1.1. 백엔드 등급 검증 로직 (1단계 적용)
* 링크 생성 API (`POST /api/links`) 호출 시, 요청 회원의 `subscription_tier`를 조회합니다.
* 무료 회원이 4번째 링크를 만들려고 하거나 픽셀 ID를 포함하여 요청하는 경우 `400 Bad Request` 에러를 반환합니다. (`code: LIMIT_EXCEEDED` 또는 `code: FEATURE_LOCKED`)
* **비회원(Anonymous)** 요청일 경우 픽셀 삽입 등 기능적 제약은 FREE 회원과 동일하게 차단되지만, **생성 가능 링크 수(3개 한도) 조건은 예외 처리되어 제한 없이 생성 가능**합니다. 대신 생성 즉시 시스템 설정 기간(일)에 기반한 만료일(`expired_at`)이 저장되며, 매일 자정 배치 스케줄러에 의해 자동으로 영구 파기됩니다.

## 2. 광고 수익화 모델 - 마케터 본인 광고 코드 매핑 (3단계: User Ad Tag Integration)

대량의 트래픽을 유발하는 전문 마케터 및 콘텐츠 창작자가 본인의 구글 광고 코드를 연동하여 발생하는 광고 수익을 100% 직접 가져가되, 플랫폼은 해당 연동 기능을 유료 플랜(PRO 등급 이상)의 핵심 스펙으로 제공하여 안정적인 정기 구독료 수익을 창출하는 모델입니다.

### 2.1. 작동 프로세스
1. 마케터가 링크 생성 시 자신의 **Google Publisher ID** 및 **Ad Slot ID**를 설정에 등록합니다. (유료 등급 권한)
2. 제삼자가 해당 단축 링크 접속 시, 5초 대기 화면(`redirect.html`)에 마케터 본인의 구글 광고 배너가 실시간 렌더링(동적 노출)됩니다.
3. 발생한 모든 광고 조회/클릭 수익은 플랫폼을 거치지 않고 **구글 애드센스로부터 마케터 본인의 계정으로 즉시 100% 직결**됩니다.

### 2.2. 어뷰징(부정클릭) 및 정산 책임 리스크 대응
* **어뷰징 위험 전이**: 매크로나 비정상 트래픽 어뷰징이 시도되더라도 플랫폼의 광고 계정 정지 리스크가 원천 차단되며, 부정 클릭을 유발한 해당 마케터 본인의 애드센스 계정이 구글 규제에 의해 제재됩니다.
* **정산 및 세무 관리 제거**: 플랫폼 운영자가 복잡한 일별 클릭당 단가(CPC) 배분 로직, FDS 부정 클릭 추적 모니터링 엔진, 세무 원천세(3.3%) 대행 처리를 직접 수행할 필요 없이 SaaS 도구 서빙에만 집중할 수 있게 설계되었습니다.

## 3. 페이월 콘텐츠 잠금 수수료 모델 (4단계: Paywall & Locks)

디지털 파일(PDF, 쿠폰 번호, 비공개 주소 등)을 링크 뒤에 잠금 처리해두고 결제 완료자에게만 리다이렉트해 주는 기능입니다.

### 3.1. 요금 구조 및 결제 흐름
1. 링크 소유자가 결제 금액(`price`)을 지정하여 페이월 링크를 생성합니다. (`isPaywalled = true`)
2. 다른 접속자가 해당 단축 링크 접속 시, 중간 페이지에 토스페이먼츠 등의 PG 결제 위젯이 뜹니다.
3. 결제가 성공하면 결제 확인 토큰이 발급되고, 세션이 인증된 접속자에 한해서만 백엔드가 목적지 URL로 최종 리다이렉트합니다.

### 3.2. 플랫폼 중개 수수료
* **수수료율**: **총 결제 금액의 5%** (결제 대행사 PG 수수료 약 3% 내외는 링크 소유자 부담)
* **정산 흐름**:
  * 결제 건당 5%의 수수료를 즉시 공제한 후, 나머지 금액이 링크 소유자의 플랫폼 예치금으로 쌓입니다.
  * 예치금은 주간 단위 혹은 월간 단위로 소유자가 등록한 정산용 계좌로 지급됩니다.
